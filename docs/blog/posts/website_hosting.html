Setting Up GitHub Pages and Cloudflare

Setting up GitHub Pages and Cloudflare involves a few key steps. Here's a detailed guide:

  1. Create a repository on GitHub: First, you need to create a repository on GitHub and upload your website files. Make sure to name the repository as `username.github.io`, where `username` is your GitHub username.
  2. Set up a CNAME record on Cloudflare: Log in to your Cloudflare account and select your domain. Go to the DNS settings and add a new record. Select 'CNAME' as the type, enter 'www' as the name, and `username.github.io` as the domain name. Replace `username` with your GitHub username.
  3. Configure GitHub Pages: Go back to your GitHub repository and navigate to the settings. Scroll down to the GitHub Pages section and enter your custom domain in the 'Custom domain' field. Save the changes.
  4. Configure Cloudflare settings: In your Cloudflare account, go to the SSL/TLS settings and set the SSL mode to 'Full'. Then, go to the 'Edge Certificates' tab and enable 'Always Use HTTPS' and 'Automatic HTTPS Rewrites'.
